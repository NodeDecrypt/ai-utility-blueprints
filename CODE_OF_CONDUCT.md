# Node Decrypt Community Code of Conduct

## Our goal

Node Decrypt Community is meant to be a welcoming, practical, and respectful place for people to share blueprints, tools, examples, and ideas.

We want this community to be:

- beginner-friendly
- respectful
- constructive
- collaborative
- safe for public contribution

---

## Expected behavior

Please do your best to:

- be respectful and patient
- assume good intent first
- give constructive feedback
- keep discussions practical and clear
- help newer contributors learn
- accept feedback gracefully
- focus on improving the work, not attacking the person
- keep contributions public-safe and community-safe

---

## Unacceptable behavior

The following behavior is not allowed:

- harassment or personal attacks
- insults, mockery, or bullying
- discrimination or hateful conduct
- sexualized language or unwanted sexual attention
- repeated aggressive or hostile behavior
- doxxing or sharing private information
- posting malicious, unsafe, or deceptive content
- sharing secrets, tokens, credentials, private auth data, or internal-only materials
- intentionally submitting misleading or harmful community resources

---

## Community content rules

Because this is a public repo, contributors must not publish:

- API keys
- tokens
- cookies
- session files
- private URLs intended for internal-only use
- selectors or automation details that should stay private
- hidden private prompt assets
- confidential internal documents
- any private or unsafe materials that do not belong in a public community repo

If you are unsure whether something is safe to publish, do not publish it.

---

## Review and feedback standards

When reviewing contributions:

- explain what should change
- be specific
- be respectful
- suggest improvements clearly
- avoid rude or dismissive language

Good review behavior includes:

- asking for clarification politely
- pointing to naming or schema rules clearly
- suggesting cleaner structure
- helping contributors fix errors

---

## Maintainer responsibilities

Maintainers are expected to:

- apply rules fairly
- protect contributors from harassment
- keep the repo aligned with its public-safe purpose
- remove clearly unsafe or inappropriate content
- guide contributors toward better structure when possible

Maintainers may edit, remove, reject, or restrict content that violates this code of conduct.

---

## Scope

This code of conduct applies to:

- GitHub issues
- pull requests
- discussions
- review comments
- repo documentation
- examples and submitted files
- other community interactions related to this repo

---

## Reporting

If you see behavior that violates this code of conduct, report it to the maintainers through the appropriate GitHub channel or contact method used by the project.

When reporting, include:

- what happened
- where it happened
- links or screenshots if available
- any other useful context

Please report in good faith.

---

## Enforcement

Maintainers may take action including:

- warning a contributor
- asking for edits
- hiding or removing content
- closing conversations
- rejecting a pull request
- temporarily or permanently banning a contributor from the project space

The response should match the severity and pattern of the behavior.

---

## Final note

This community is built for sharing useful work in a clear and respectful way.

Be kind, be clear, and help make Node Decrypt Community easier for the next person.
