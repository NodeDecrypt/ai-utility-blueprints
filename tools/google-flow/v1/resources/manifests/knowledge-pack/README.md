# Knowledge Pack

This folder is a manifest resource for the `google_flow` tool.

It is used to hold grouped reference files that may support the tool during usage.

---

## What this manifest is for

Use this folder when the tool may benefit from a set of supporting files that provide reusable context or guidance.

Examples:

- brand rules
- visual direction notes
- format guides
- campaign references
- design examples
- public-safe instruction documents

This is a grouped reference pack, not a blueprint step by itself.

---

## What can go in this folder

Examples of suitable files:

- `.md`
- `.txt`
- `.pdf`
- image reference files if appropriate
- small public-safe supporting documents

Example content:

- brand style guide
- layout reference notes
- image direction guide
- campaign summary
- visual do/don’t examples

---

## How a human runner should use it

A human runner can treat this folder as:

- a reusable support pack for the tool
- a reference set to review before generation
- a place to keep grouped public-safe supporting materials

If a blueprint uses this tool and the runner wants stronger consistency, files from this folder may help provide reusable context.

---

## Good practice

Try to keep the files:

- clearly named
- relevant to the tool’s purpose
- reusable across multiple blueprints when possible
- public-safe
- easy to browse

Helpful file names:

- `brand-style-guide.pdf`
- `visual-direction.md`
- `layout-notes.md`
- `campaign-reference.pdf`

---

## What not to include

Do **not** include:

- private documents
- confidential client files
- internal-only system instructions
- API keys, tokens, or credentials
- hidden automation details
- anything unsafe to publish publicly

This repo is public and community-facing.

---

## Suggested folder use

A typical folder might look like this:

```text
knowledge-pack/
├── README.md
├── brand-style-guide.pdf
├── visual-direction.md
├── layout-notes.md
└── campaign-reference.pdf
```
